Merge vs Rebase

The most commonly confused pair in Git interviews.

Merge

Keeps full history. Adds an extra commit that ties both branches together. History looks like a tree.

Rebase

Rewrites history into one straight line. Cleaner, but risky if others already pulled the old commits.

Undo Commands

Know which one to reach for — this gets asked almost every time.

SituationCommand
Undo unstaged changes in a filegit restore <file>
Unstage a file, keep the changesgit reset <file>
Undo last commit, keep changesgit reset --soft HEAD~1
Undo last commit, discard changesgit reset --hard HEAD~1
Undo a commit safely on a shared branchgit revert <hash>
Save work-in-progress, switch branch cleanlygit stash
reset = rewrites history (risky if shared) · revert = adds a new commit that undoes one (safe)
